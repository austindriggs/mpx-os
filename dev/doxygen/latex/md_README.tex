Prep your linux environment by running the following commands\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{sudo\ apt\ update}
\DoxyCodeLine{sudo\ apt\ install\ -\/y\ clang\ make\ nasm\ git\ binutils-\/i686-\/linux-\/gnu\ qemu-\/system-\/x86\ gdb}

\end{DoxyCode}


Then run {\ttfamily make} and {\ttfamily ./mpx.sh}.

For more information, either run the {\ttfamily help} command inside of Macaroni\+OS, or consult the \href{doc/USER-GUIDE.pdf}{\texttt{ doc/\+USER-\/\+GUIDE.\+pdf}}.\hypertarget{md_README_autotoc_md2}{}\doxysection{\texorpdfstring{CONTRIBUTING}{CONTRIBUTING}}\label{md_README_autotoc_md2}
After making changes, running {\ttfamily version} will show that your working directory is \textquotesingle{}dirty\textquotesingle{}. This simply means that you have uncommitted changes.

Ensure you have checked out the correct branch and pulled its latest changes. Stage/add the relevant files before committing them.

Now you can run {\ttfamily make clean} and {\ttfamily make} again, run {\ttfamily ./mpx.sh}, and finally run {\ttfamily version} to see your latest commit hash and showing that your working directory is \textquotesingle{}clean\textquotesingle{}.

When you\textquotesingle{}re done, add your contributions to \href{dev/CONTRIBUTIONS.docx}{\texttt{ dev/\+CONTRIBUTIONS.\+docx}} and save it as \href{/doc/CONTRIBUTIONS.pdf}{\texttt{ doc/\+CONTRIBUTIONS.\+pdf}}.\hypertarget{md_README_autotoc_md3}{}\doxysection{\texorpdfstring{DOXYGEN}{DOXYGEN}}\label{md_README_autotoc_md3}
